Getting Started

Requirements

The only hard requirement is Java 7+. Particular processors may have additional requirements.

Download

Download the latest release from GitHub.

Configuration

The Cantaloupe .zip file bundles a sample configuration file, named cantaloupe.properties.sample. Do not edit this file — instead, create a copy and edit that. (These instructions assume a name of cantaloupe.properties, but any name will work.)

To get started quickly, just set FilesystemResolver.BasicLookupStrategy.path_prefix to a folder containing some images, and leave all the other options alone.

Running

From the command prompt:

$ java -Dcantaloupe.config=/path/to/cantaloupe.properties -Xmx2g -jar Cantaloupe-x.x.x.jar

-Xmx2g tells the JVM to use up to 2GB of heap space (RAM). Without this argument, some JVMs will default to a much smaller amount, leading to OutOfMemoryErrors.

Note: on OS X, a "java" icon will appear in the Dock. This can be prevented by adding the -Djava.awt.headless=true argument to the command.

Assuming the http.port option has not been changed, it is now running at: http://localhost:8182/. To test, assuming you have an image named image.tif:

http://localhost:8182/iiif/2.0/image.tif/full/pct:20/0/default.jpg

Upgrading

Upgrading is usually just a matter of downloading a new version and running it. Since instances are self-contained, new versions can run happily alongside existing ones, with each using its own config file. Sometimes there are backwards-incompatible changes to the configuration file structure, though, so check below to see if there is anything more to be done.

1.2 to 2.0

Note that the Image API 2.0 endpoint, which used to be located at /iiif, has moved to /iiif/2.0.

Likewise, the landing page, which used to be located at /iiif, has moved to /.

  • Add the following new keys from the sample configuration:
    • max_pixels
    • endpoint.iiif.1.1.enabled
    • endpoint.iiif.2.0.enabled
    • FilesystemResolver.lookup_strategy
    • FilesystemResolver.ScriptLookupStrategy.script
    • HttpResolver.lookup_strategy
    • HttpResolver.ScriptLookupStrategy.script
    • JdbcResolver.connection_timeout
    • JdbcResolver.max_pool_size
    • cache.server.purge_missing
    • JdbcCache.connection_timeout
    • JdbcCache.max_pool_size
  • Rename the following keys:
    • FilesystemResolver.path_prefix to FilesystemResolver.BasicLookupStrategy.path_prefix
    • FilesystemResolver.path_suffix to FilesystemResolver.BasicLookupStrategy.path_suffix
    • HttpResolver.url_prefix to HttpResolver.BasicLookupStrategy.url_prefix
    • HttpResolver.url_suffix to HttpResolver.BasicLookupStrategy.url_suffix
  • Delete the following obsolete keys:
    • log.application.ConsoleAppender.pattern
    • log.application.FileAppender.pattern
    • log.application.RollingFileAppender.pattern
  • Because the cache naming format has changed, it will be necessary to purge your cache before using this version.

1.1 to 1.2

  • Add the http.content_disposition key from the sample configuration.
  • Add the AmazonS3Resolver.* keys from the sample configuration.
  • Add the FfmpegProcessor.path_to_binaries key from the sample configuration.
  • Add the cache.server.resolve_first key from the sample configuration.
  • Remove the generate_https_links key from your configuration file, and add the base_uri key from the sample configuration.

1.0.x to 1.1

  • Add the cache.client.enabled key from the sample configuration.

1.0-beta3 to 1.0

  • Add the logging.* keys from the sample configuration.
  • Add the generate_https_links key from the sample configuration.
  • Add the FfmpegProcessor.path_to_binaries key from the sample configuration.
  • Rename HttpResolver.username to HttpResolver.auth.basic.username in the configuration.
  • Rename HttpResolver.password to HttpResolver.auth.basic.secret in the configuration.

1.0-beta2 to 1.0-beta3

  • Rename the cache key in the configuration to cache.server.
  • Add the cache.client.* keys from the sample configuration.
  • Add the http.auth.basic* keys from the sample configuration.
  • Add the KakaduProcessor keys from the sample configuration.
  • Change any ImageIoProcessor references to Java2dProcessor in the configuration.
  • Add the Java2dProcessor.tif.reader key from the sample configuration.

1.0-beta1 to 1.0-beta2

  • Add the cache and FilesystemCache.* keys from the sample configuration.
  • Add the FilesystemResolver.path_separator and HttpResolver.path_separator keys from the sample configuration.

Getting Help

If you find yourself stuck on anything, please file a GitHub issue. Include a description of the problem as well as any other relevant information, such as:

  • Your Cantaloupe version
  • Your JVM's heap size (this is printed on the landing page as well as in the first few lines of console output)
  • The request URL
  • The effective processor, resolver, and cache
  • Log output in debug mode
  • Configuration options with sensitive information removed
